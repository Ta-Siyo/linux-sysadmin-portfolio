##Objectives
- Install SSH Server
- Configure SSH keys
- Disable password authentication

## Security Improvements
- Reduced brute-force attack risk
- Enforced key-based authentication

## Verification
- SSH service running
- Password login disabled
- Key authentication working

## SSH Port Configuration

The SSH service remained on the default port (22).

Changing the SSH port was considered but not implemented in this lab 
environment to avoid disrupting remote access while learning SSH hardening.

Other security measures implemented:
- Key-based authentication
- Restricted SSH access
- Firewall configuration
